## What this repo is
LyraSec AI — an agent-native application-security platform for AI-built software. Core loop: **Target → Scan → Verified Finding → Fix PR → Retest → Report**. Turborepo + pnpm monorepo: `apps/web` (Next.js 16 App Router, React 19, TS 6), `apps/worker` (scan engine worker with BullMQ queue, preflight checks, engine runner, finding persister), and `packages/{auth,config,db,integrations,logger,types,ui}`. Stack details, architecture, conventions, and a file-by-file map: **`codebase.md`** (start at §17 for Batch 1 changes, §18 for UI/UX premium upgrade + deep code review, §19 for Batch 2 remainder + Batch 3 design contracts + deep code review fixes, §20 for Round-2 hardening, §21 for worker/engine + review fixes, §22 for Batch 4 features + code review fixes).

## Landmines — do NOT re-break these
- **Prisma migration drift is reconciled.** A reconciling migration (`20260705095000_batch3_missing_tables_columns`) creates all missing tables (`ApiKey`, `Retest`, `OnboardingState`) and adds all missing columns/indexes/constraints. CI now runs `prisma migrate diff --exit-code` to catch future drift. **Verify on a fresh DB before prod deploy:** `prisma migrate reset && prisma migrate deploy` should reproduce `schema.prisma` exactly.
- **Prisma model sets must match schema columns.** `packages/db/src/scoping.ts` `SOFT_DELETE_MODELS` (19) may contain *only* models with a `deletedAt` column; `WORKSPACE_SCOPED_MODELS` (17) only models with a `workspaceId` column. Adding a model without the column makes the client extension inject a non-existent field → **Prisma throws on every read**. (Real bug once: soft-delete set included `WorkspaceMember`/`CredentialSet`/`AuditLog`/`Retest`, crashing `getWorkspaceMembership`.)
- **Workspace scoping is auto-active.** `requireWorkspaceAccess` calls `setWorkspaceContext(workspaceId)` (AsyncLocalStorage). `WorkspaceMember` (cross-workspace switcher) and `OnboardingState` (per-user) are deliberately excluded from auto-scoping — do not add them. Never reintroduce a module-level global for workspace context (cross-request tenant leak).
- **Postgres RLS is live.** RLS enabled on all 17 workspace-scoped tables with permissive (no context = allow all, backward compat) + strict (context set = only matching `workspaceId`) policies. Use `withWorkspaceRLS(workspaceId, fn)` from `@lyrashield/db` to wrap queries that need DB-level isolation. The helper uses `SET LOCAL` inside a transaction (connection-safe with Prisma pooling). CI validates the migration on Postgres 16.
- **Workflow files need special GitHub permission.** The Hyperagent GitHub app lacks `Workflows: write`, so agents using it get 403 writing `.github/workflows/*`. Route CI/workflow edits to a user-credentialed agent (e.g. Codex) or the repo owner.
- **Rename is partial.** Repo is `ecryptoguru/lyrasec-ai`; product is migrating LyraShield → LyraSec. **Do NOT rename** `@lyrashield/*` package scopes or `LYRASHIELD_*` env vars without founder sign-off (trademark clearance open). Keep everything domain-agnostic (`NEXT_PUBLIC_APP_URL`); public domain is undecided.
- **A4 follow-up:** store the numeric GitHub `installationId` on `Target` and match on it (webhook currently matches by `repoFullName` owner-prefix).
- **Schedule.target FK migration exists.** Migration `20260706010000_schedule_target_fk` adds the `Schedule_targetId_fkey` foreign key. Do not drop or recreate it. If you add a new migration that touches the `Schedule` table, ensure it doesn't conflict with this constraint.
- **Worker notifications have no `userId`.** Notifications created by the worker (`notifyScanCompleted`, `notifyScanFailed`, `notifyCriticalFinding`) are workspace-level — they do not set `userId`. The notifications list API must not filter by `userId` by default, or worker-created notifications will be invisible. Use the optional `userId` query param only when user-specific filtering is needed.
- **`createAndSendNotification` is the shared notification helper.** Located in `packages/db/src/notification-service.ts`. It accepts a `sendFn` callback to avoid circular dependencies between `@lyrashield/db` and `@lyrashield/integrations`. Both the worker (`apps/worker/src/notifications.ts`) and the create-pr API route use it. Do not duplicate the create+send loop — use this helper.

## Conventions (see codebase.md §14 for the full list)
- **Never push to `main` directly** — branch + PR. CI must pass.
- **Verify before "done":** `pnpm lint && pnpm typecheck && pnpm test && pnpm build`. For any security-relevant change, add a test proving the control holds.
- **Auth imports:** `@lyrashield/auth` (client) vs `@lyrashield/auth/server` (server). Scope every workspace query by `workspaceId`. Validate all input with Zod. Write audit logs on sensitive ops.
- **Logging:** use `@lyrashield/logger` — it redacts sensitive keys, but still never log raw secrets/tokens deliberately.
- **Honest positioning** in any public-facing text/comments/sample reports: no "only we…" claims, no benchmark/accuracy numbers, no public pricing, never name the forked scan engine publicly.
- **High-risk zones (slow down, add tests):** the scan sandbox/worker, auth/RBAC/tenant isolation, the MCP approval gates, secrets handling, billing/metering, Prisma migrations, the CI GitHub Action.
- **UI conventions:** Use shared components from `@lyrashield/ui` (Button, Card, Badge, Input, Textarea, Select, FormField, EmptyState, Spinner, LoadMore, GithubIcon). Use `rounded-lg`/`rounded-xl` (not `rounded-md`). Use premium utility classes from `globals.css` (`.glass`, `.gradient-primary`, `.gradient-hero`, `.text-gradient`, `.shadow-premium`, `.shadow-card-hover`, `.shadow-primary-glow`) — all have dark mode variants. All decorative icons must have `aria-hidden="true"`. Use typed API helpers from `@/lib/api-client` (`apiGet`, `apiPost`, `apiPatch`, `apiDelete`, `apiGetPaginated`) instead of raw `fetch` in client components. Use React `cache()` wrappers from `@/lib/cache` for server-side data fetching deduplication.
