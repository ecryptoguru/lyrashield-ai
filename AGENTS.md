## Landmines вЂ” do NOT re-break these
- **рџ”ґ Prisma migrations are DRIFTED from `schema.prisma` (latent P0 on deploy).** Only 2 migrations exist; they do NOT create `ApiKey`/`OnboardingState`/`Retest` or many columns/indexes present in `schema.prisma` (the schema was synced to dev via `prisma db push`). So `prisma migrate deploy` on a **fresh** DB (CI/prod) yields a database that mismatches the client в†’ onboarding/apikey/retest queries fail at runtime. **Until a reconciling migration is generated (`prisma migrate dev`, then verify `migrate reset`+`deploy` reproduces the schema), use `prisma db push` for local dev, not `migrate deploy`.** Fixing this + adding a CI drift-check is the top open task (PRD В§B13.7 R-C/R-F).
- **Prisma model sets must match schema columns.** `packages/db/src/scoping.ts` `SOFT_DELETE_MODELS` (19) may contain *only* models with a `deletedAt` column; `WORKSPACE_SCOPED_MODELS` (17) only models with a `workspaceId` column. Adding a model without the column makes the client extension inject a non-existent field в†’ **Prisma throws on every read**. (Real bug once: soft-delete set included `WorkspaceMember`/`CredentialSet`/`AuditLog`/`Retest`, crashing `getWorkspaceMembership`.)
- **Workspace scoping is auto-active.** `requireWorkspaceAccess` calls `setWorkspaceContext(workspaceId)` (AsyncLocalStorage). `WorkspaceMember` (cross-workspace switcher) and `OnboardingState` (per-user) are deliberately excluded from auto-scoping вЂ” do not add them. Never reintroduce a module-level global for workspace context (cross-request tenant leak).
- **Postgres RLS is live.** RLS enabled on all 17 workspace-scoped tables with permissive (no context = allow all, backward compat) + strict (context set = only matching `workspaceId`) policies. Use `withWorkspaceRLS(workspaceId, fn)` from `@lyrashield/db` to wrap queries that need DB-level isolation. The helper uses `SET LOCAL` inside a transaction (connection-safe with Prisma pooling). CI validates the migration on Postgres 16.
- **Workflow files need special GitHub permission.** The Hyperagent GitHub app lacks `Workflows: write`, so agents using it get 403 writing `.github/workflows/*`. Route CI/workflow edits to a user-credentialed agent (e.g. Codex) or the repo owner.
- **Rename is partial.** Repo is `ecryptoguru/lyrasec-ai`; product is migrating LyraShield в†’ LyraSec. **Do NOT rename** `@lyrashield/*` package scopes or `LYRASHIELD_*` env vars without founder sign-off (trademark clearance open). Keep everything domain-agnostic (`NEXT_PUBLIC_APP_URL`); public domain is undecided.
- **A4 follow-up:** store the numeric GitHub `installationId` on `Target` and match on it (webhook currently matches by `repoFullName` owner-prefix).

## Conventions (see codebase.md В§14 for the full list)
- **Never push to `main` directly** вЂ” branch + PR. CI must pass.
- **Verify before "done":** `pnpm lint && pnpm typecheck && pnpm test && pnpm build`. For any security-relevant change, add a test proving the control holds.
- **Auth imports:** `@lyrashield/auth` (client) vs `@lyrashield/auth/server` (server). Scope every workspace query by `workspaceId`. Validate all input with Zod. Write audit logs on sensitive ops.
- **Logging:** use `@lyrashield/logger` вЂ” it redacts sensitive keys, but still never log raw secrets/tokens deliberately.
- **Honest positioning** in any public-facing text/comments/sample reports: no "only weвЂ¦" claims, no benchmark/accuracy numbers, no public pricing, never name the forked scan engine publicly.
- **High-risk zones (slow down, add tests):** the scan sandbox/worker, auth/RBAC/tenant isolation, the MCP approval gates, secrets handling, billing/metering, Prisma migrations, the CI GitHub Action.
- **UI conventions:** Use shared components from `@lyrashield/ui` (Button, Card, Badge, Input, Textarea, Select, FormField, EmptyState, Spinner, LoadMore, GithubIcon). Use `rounded-lg`/`rounded-xl` (not `rounded-md`). Use premium utility classes from `globals.css` (`.glass`, `.gradient-primary`, `.gradient-hero`, `.text-gradient`, `.shadow-premium`, `.shadow-card-hover`, `.shadow-primary-glow`) вЂ” all have dark mode variants. All decorative icons must have `aria-hidden="true"`. Use typed API helpers from `@/lib/api-client` (`apiGet`, `apiPost`, `apiPatch`, `apiDelete`, `apiGetPaginated`) instead of raw `fetch` in client components. Use React `cache()` wrappers from `@/lib/cache` for server-side data fetching deduplication.
